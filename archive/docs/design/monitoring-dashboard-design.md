# S001-Pro 监控面板 - Design Doc

> 生成时间: 2026-04-07  
> 通过: `/office-hours` 流程

---

## 1. 问题定义

### 现状
- 需要 SSH 到服务器查看策略状态
- 无法随时了解策略运行情况
- 无法方便地向他人展示项目状态

### 目标
构建一个**专业级实时监控面板**，实时展示 S001-Pro 策略运行状态，支持分享给他人查看。

### 成功标准
- [ ] 数据实时刷新（<5秒延迟）
- [ ] 专业交易面板体验
- [ ] 手机完美适配
- [ ] 密码登录保护

---

## 2. 用户需求

### 主要用户
- **自己**: 全面管理，需要所有数据和操作入口
- **朋友/投资者**: 只能查看，不能操作

### 权限设计
| 功能 | 自己 | 朋友 |
|-----|------|------|
| 查看实时数据 | ✅ | ✅ |
| 查看历史记录 | ✅ | ✅ |
| 暂停/启动策略 | ✅ | ❌ |
| 修改配置 | ✅ | ❌ |

## 3. 功能范围

### Phase 1: 专业核心（1周交付）

- ✅ 收益曲线图（日/周/月切换）

  - 时间戳
  - 级别（INFO/WARNING/ERROR）

  - 筛选功能（全部/订单/错误）

- ✅ 在线/离线状态

#### 3.3 技术特性
- ✅ WebSocket 实时推送 或 5秒轮询
- ✅ 手机响应式设计（像专业交易APP）
- ✅ 密码登录保护
- ✅ 数据自动刷新

### Phase 2: 专业增强（第2周）
- 历史订单查询（带筛选、分页）

## 4. 技术方案要点

### 4.1 架构设计
```
┌─────────────┐     WebSocket/HTTP     ┌─────────────┐
│   前端      │ ◄────────────────────► │   后端      │
│  React/Vue  │                        │  Python     │
│  实时监控   │                        │  Flask/FastAPI
└─────────────┘                        └──────┬──────┘
                                              │
                    ┌─────────────────────────┼─────────────────────────┐
                    │                         │                         │
                    ▼                         ▼                         ▼
            ┌─────────────┐          ┌─────────────┐          ┌─────────────┐
            │  SQLite     │          │  Binance    │          │  System     │
            │  交易数据   │          │  API        │          │  监控       │
            └─────────────┘          └─────────────┘          └─────────────┘
```

### 4.2 实时数据方案
- **方案A**: WebSocket（推荐）
  - 服务器主动推送
  - 延迟 < 1秒
  - 复杂度中等

- **方案B**: HTTP轮询（备选）
  - 5秒轮询一次
  - 延迟 3-5秒
  - 实现简单

### 4.3 数据源
- 交易数据: SQLite 数据库（~/S001-Pro/data/）
- 实时价格: Binance API
- 服务器状态: SSH/systemd status
- 日志: journalctl 或本地日志文件

### 4.4 安全考虑
- 密码登录（bcrypt加密）
- JWT token 认证
- 只读权限给朋友账号
- API限流防刷

## 6. 技术选型

### 6.1 前端
- **推荐**: React + TypeScript + Tailwind CSS
- **备选**: Vue 3 + TypeScript
- **图表**: Chart.js 或 ECharts
- **实时**: Socket.io 或原生 WebSocket

### 6.2 后端
- **推荐**: Python FastAPI
- **备选**: Flask + Flask-SocketIO
- **数据库**: SQLite (只读访问 S001-Pro 数据)
- **认证**: JWT + bcrypt

### 6.3 部署
- 与 S001-Pro 同服务器部署
- Nginx 反向代理
- 自动启动 (systemd)

## 8. 风险与注意事项

### 8.1 风险
- **数据安全**: 交易数据敏感，需要密码保护
- **性能**: 实时查询可能影响策略性能
- **稳定性**: WebSocket 断线重连处理

### 8.2 规避方案
- 数据库只读访问
- 使用缓存（Redis/内存）
- 前端降级到轮询
- 限流保护
