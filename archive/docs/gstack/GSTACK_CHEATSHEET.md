# gstack 速查卡

## 30秒上手

```
/office-hours    定义问题 (新功能第一步)
/plan-eng-review 技术方案 (编码前)
/review          代码审查 (提交前)
/qa              测试验证 (部署前)
/ship            一键发布 (一切就绪)
/retro           每周复盘
```

## 何时使用

```
有新想法      → /office-hours
设计技术方案   → /plan-eng-review
写完代码      → /review
准备上线      → /qa
代码就绪      → /ship
周末复盘      → /retro
遇到bug      → /investigate
危险操作      → /careful
```

## S001-Pro 专用

```
/plan-risk-review   风控方案评审
/review-s001        S001代码审查
/qa-s001           回测QA验证
```

## 核心原则

**Boil the Lake**: AI让完整实现变得便宜，不要走捷径。

**流程优先**: 定义 → 设计 → 编码 → 审查 → 测试 → 发布

**强制检查**: 任何代码修改必须经过 review 才能提交
